# 競馬単勝オッズ予測エンジンの構築プロセス詳細ドキュメント (HOWTOMAKE)

本ドキュメント群では、競馬の単勝オッズを予測する機械学習モデルを、基礎的な回帰分析から始まり、JRAの配当ロジックの組み込み、数学的なキャリブレーション、そして物理的・心理的特徴量の特化追加などを経て、徐々に高度化させていった全19サイクルの開発・検証プロセスを極めて詳細に記述します。

各段階の詳しいアプローチ、数式、および実装の意図については、以下の各章のリンクから詳細ドキュメントをご参照ください。

---

## 目次 (Index)

### 📊 [序章：競馬オッズのモデリングにおける本質的課題とアプローチ](HOWTOMAKE_0.md)
競馬のオッズ形成メカニズムと、JRA（日本中央競馬会）の単勝払戻率に基づくパリミュチュエル方式の基本定理、および本プロジェクトの最終目標について解説します。 **（公式の参照URL情報を含みます）** 

### 🐎 [第一段階：基礎属性によるベースライン構築 (Cycle 1-3)](HOWTOMAKE_1.md)
馬齢や斤量などの静的プロファイルのみを用いた初期の試みと、オッズのロングテール分布に起因する平均二乗誤差（MSE）の破綻、および「騎手」と「過去実績」要素の第一歩について。

### 🔢 [第二段階：JRA払戻率 80% ルール（確率的制約）の導入 (Cycle 4)](HOWTOMAKE_2.md)
オッズに対する直接回帰を放棄し、予測対象を「支持率（Probability）」にスワップして0〜0.8の大元に正規化（Normalization）する根本的な数学モデルの改革について。

### ⏱️ [第三段階：コース適性と物理指数への還元論の精緻化 (Cycle 5-7)](HOWTOMAKE_3.md)
「中山競馬場巧者」の抽出や、最高平均速度（m/s）および上がり3ハロン平均タイムといった、距離に依存しない物理単位での馬のエンジン性能指標の導入について。

### 🧠 [第四段階：人間要素と市場心理・バイアスの深層モデリング (Cycle 8-10)](HOWTOMAKE_4.md)
「勝ち上がりコンビ」の評価、「乗り替わり」による陣営の勝負気配の推測、および「前走の着順・人気（新聞の文字面）」がいかに群衆心理に直結するかについての行動モデリング。

### 📈 [第五段階：対数領域への写像とキャリブレーション問題 (Cycle 11-13)](HOWTOMAKE_5.md)
支持率空間の微細なブレを学習させるための「対数支持率（Log-Probability）」の導入と、行動経済学に基づく「本命・大穴バイアス（Favorite-Longshot Bias）」の数学的写像補正パラメーター $k$ の設置について。

### ⚙️ [第六段階：時系列検証とハイパーパラメータの最適化 (Cycle 14-19)](HOWTOMAKE_6.md)
データリークを防ぐ厳格な時系列時差バックテストの実施と、オーシャンステークスに特化した心理バイアスの逆転現象を示す最適解 $k=1.2$ の発見およびその意味論的解釈。

### 🚀 [第七段階：実運用へ向けたアーキテクチャ再構築と永続化 (Final)](HOWTOMAKE_7.md)
モデル予測を「実力テスト」から「リアルタイムな実弾予想エンジン」へ昇華させるための、Parquetファイル化によるキャッシュアーキテクチャの導入と計算コスト削減のリファクタリング過程。

## 付録: 学習スクリプト（`train/`）とドキュメントの対応マップ
GitHubリポジトリ内の `train/` ディレクトリに収録されている主要な実行スクリプトと、本ドキュメント群における各開発段階の対応関係は以下の通りです。

| スクリプト名 | 該当ドキュメント | 推移と機能概要 |
|:---|:---|:---|
| `train_v1.py` 〜 `v3.py` | [第一段階](HOWTOMAKE_1.md) | 基礎属性のみでのMSEによる初期学習（オッズ直接回帰の失敗）とジョッキー実績の追加 |
| `train_v4.py` 〜 `v6.py` | [第二段階](HOWTOMAKE_2.md) | オッズから支持率（Probability）予測への転換と、出走取消に対応した80%正規化層の導入 |
| `train_v7.py` 〜 `v9.py` | [第三〜第四段階](HOWTOMAKE_3.md), [HOWTOMAKE_4.md] | 物理的指標（m/s等）や、乗り替わり・コンビ実績などの市場心理・行動バイアス指標の実装 |
| `train_v10.py` 〜 `v12_heavy.py` | [第五段階](HOWTOMAKE_5.md) | 目的変数を自然対数領域（Log-Probability）へ写像変換した高精度学習層の構築 |
| `train_v13_final.py` | [第五段階](HOWTOMAKE_5.md) | 本命・大穴バイアスを補正するための「キャリブレーション層（パラメータ $k$）」の組み込み |
| `train_v14_*.py`, `v15_*.py` | [第六段階](HOWTOMAKE_6.md) | 未来のデータを含めない「完全時系列ローリング検証」とOptunaを用いたハイパーパラメータ最適化 |
| `train_v16_persist.py` | [第七段階](HOWTOMAKE_7.md) | I/Oボトルネックを解消するためのParquet永続化など、実運用向けシステムアーキテクチャの構築 |
| `train_v19_final_best.py` | [全章の集大成](HOWTOMAKE.md) | 全19サイクルの知見（特徴量・対数空間・正規化・k値補正）を統合した最終確定版学習パイプライン |
| `optimize_calibration.py` | [第六段階](HOWTOMAKE_6.md) | オーシャンSなどの過去データから、最も誤差の少なくなる最適キャリブレーション値（$k=1.2$等）を探索抽出するツール |
| `backtest_ocean_stakes.py` | [第六段階](HOWTOMAKE_6.md) | 特定の重賞レースに絞って、過去5年分の回収率や期待値の推移を通しでシミュレーションする検証スクリプト |
| `common.py` | [第七段階](HOWTOMAKE_7.md) | 複数のバージョンの機械学習スクリプトから呼び出される、共通の特徴量生成・評価系関数モジュール |

**結論サマリー** ：
本単勝オッズ予測エンジンの開発は、単なる表形式（Tabular）データの機械学習への入力作業にとどまらず、競馬のパリミュチュエルシステムの数理的逆算から始まり、速度の物理計算、騎乗変更に隠された心理分析、そして行動経済学のバイアス補正に至るまで、極めて学際的な知見をコード設計（LightGBM）に統合するプロセスでした。
最終バージョン（Cycle 19）のモデルは、十数年の数万レコードという巨大なトレンドの波に局所的なファン心理の機微を加味して、プロ予想家のコンセンサスを完全シミュレートする「最適オッズの自動生成エンジン」として完成しました。
