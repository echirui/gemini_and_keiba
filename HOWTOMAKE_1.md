# 第一段階：基礎属性によるベースライン構築 (Cycle 1-3)

本章では、プロジェクトの最初期段階にあたるベースラインモデルの構築と、それに伴って明らかになった機械学習によるオッズ予測特有の問題点について解説します。対象となるソースコードは主に `train_v1.py` から `train_v3.py` に該当します。

## 1. Cycle 1: 静的プロファイルのみでの初期アプローチ
開発の最初期段階では、変数を極力絞り込んで問題空間をシンプルにするため、予測対象を「中山競馬場・芝1200m（オーシャンステークス等の舞台）」という特定の条件のレースデータのみに限定して開始しました。

初期段階でモデル（LGBMRegressor）に投入した「静的プロファイル」特徴量は以下の通りです。
* **`horse_number`（馬番）** : レースにおける枠順。内枠有利か外枠有利かといったコース取りの有利不利の推定。
* **`handicap`（斤量）** : 出走馬が背負う負担重量。重いほど物理的に不利に働くと予想される。
* **`age`（馬齢）** : 馬の年齢。若馬の勢いや成長度、高年齢馬の衰えの指標。
* **`sex_encoded`（性別）** : 牡馬（オス）、牝馬（メス）、セン馬（去勢）などのカテゴリ変数をLabel Encodingを用いて決定木が扱える数値に変換。

**直面した課題 (MSEとロングテールの不適合)** 
初期モデルでは、オッズを直接の目的変数（$y = Odds$）として回帰モデルの学習を行いました。しかし、すぐに「オッズのロングテール分布と平均二乗誤差（MSE）の相性の悪さ」という致命的な課題に直面しました。
単勝オッズは1.1倍（超大本命）から300倍以上（超大穴）まで極端な非対称分布を描きます。LGBMが誤差を縮小しようとする際、超大穴馬の予測値が100倍から150倍に50倍ずれた場合の強烈な誤差ペナルティを打ち消そうとするあまり、実態として圧倒的に重要である「上位人気馬の1.5倍から3.0倍の誤差」を軽視してしまうという現象が発生しました。結果として有力馬のオッズ予測精度が実用に耐えないレベルで壊滅しました。

## 2. Cycle 2: ジョッキー（騎手）要素の導入
競馬には古くから「馬が7割、騎手が3割」という格言があります。誰が手綱を握るかが、馬の実力以上にファンの馬券購買心理（オッズ）に甚大な影響を与えます。
Cycle 2では、データベース上の `keibadatabase_jockey` テーブルを結合し、特定のジョッキーに対する大衆の信頼度を数値化してモデルに投入しました。

追加した特徴量：
* **`jockey_avg_odds`** : 直近4年間における該当騎手の平均オッズ。低いほど「常に人気を集めるカリスマ騎手」を意味する。
* **`jockey_win_rate`** : 直近4年間における勝率。
* **`jockey_ride_count`** : 騎乗回数。試行回数を示すことで、新人騎手の一過性の高勝率などのノイズを平滑化する目的。

この要素の導入により、「ルメール騎手や川田騎手などトップジョッキーが乗る馬であれば、馬の成績が平凡でも無条件でオッズが暴落（人気する）する」という市場のバイアスをモデルが学習し始め、現実のオッズとの乖離が抑えられました。

## 3. Cycle 3: 全期間における馬の実績評価の導入
静的なプロファイルと騎手の要素に加え、「これまでに出走した全レースでどれだけ強い競馬をしてきたか」という馬固有のポテンシャルを示す実績データを追加しました。

追加した特徴量：
* **`horse_avg_finish_pos`** : キャリア全戦における平均着順。数値が小さいほど好成績。
* **`horse_avg_popularity`** : キャリア全戦における平均人気。過去どれだけファンから支持されてきたかの履歴。
* **`horse_avg_odds`** : 全期間の平均オッズ。

これにより、モデルはやっと「この馬は過去平均して3着に来るような実力馬であり、いつも人気を集めている」というベースラインのポテンシャルを把握できるようになりました。しかし、最初の課題であった「直接オッズをMSEで予測することの限界」は数学的なアプローチなしには解決不可能であることが判明しました。

## 4. データパイプラインと未出走馬（欠損値）の扱い
本ベースライン構築において、実運用を見据えたデータパイプライン上で避けて通れなかったのが「過去のレース履歴が全く存在しない馬（新馬戦や、未勝利戦の初出走馬など）」の取扱いです。

過去の平均着順や平均オッズが存在しない馬に対して、安易に「平均値（Mean）やゼロで補完する」という手法はとりませんでした。なぜなら、競馬において「出走履歴がないこと」自体がファン心理（オッズ）に多大な不確実性バイアスを与える特異なステータスだからです。
そのため、本モデルでは履歴を持たない馬の特徴量（`horse_avg_finish_pos`等）にはすべて明示的に **欠損値（NaN）** を代入したまま学習パイプラインへ流すアプローチを選択しました。

LightGBMなどの決定木アルゴリズムは、NaNを欠損値として認識し、「NaNである場合は右の分岐（期待値がわからない未知の馬グループ）、値がある場合は左の分岐（実績馬グループ）」といったように、**「データが存在しないこと自体を意味のある分岐条件（構造的特徴）として自動的に学習」**することができます。血統データ等の代替補完を行わずとも、このアルゴリズム特性を利用することで「よくわからない新馬に対するファンの慎重なオッズ形成心理」をモデル内に表現することに成功しています。
