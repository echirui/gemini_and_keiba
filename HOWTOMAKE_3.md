# 第三段階：コース適性と物理指数への還元論の精緻化 (Cycle 5-7)

本章では、ベースラインと数学的な枠組み（払戻率の制約）が完成した段階から、特徴量エンジニアリング（Feature Engineering）を高度化していったプロセスについて解説します。
単なる「全戦績の平均」という大雑把な指標から、当日の「中山競馬場・芝1200m」という固有の舞台設定に適合した精緻なパラメータ群へとモデルを特化させました。

## 1. Cycle 5: コース・距離適性の分離抽出
初期のモデルでは「全期間の平均着順」を馬の基本能力として入力していました。しかし、競馬には「距離適性」や「コースの得手不得手」が強烈に存在します。短距離（スプリント）専用のスピード馬が、中距離（2000m）を走ってスタミナ切れで大敗した際のノイズが「平均実績」を押し下げてしてしまうケースが頻発しました。
そこで、条件を意図的に細分化した適性スコア群を個別の素性として抽出しました。

* **`horse_nakayama_avg_finish_pos`**: 中山競馬場（右回り・急坂という特殊なコースレイアウト）に限定した平均着順。中山というトリッキーな舞台の「コース巧者」を炙り出す。
* **`horse_dist_avg_finish_pos`**: 芝1200m（スプリント戦）のみに限定した戦績から算出した平均着順。純粋な短距離適性のスコア。

これにより、モデルは「他の競馬場では凡走しているが、中山1200mに限っては常に好走するスペシャリスト」を、大衆よりも早く正確に評価できるようになりました。

## 2. Cycle 6: 「速度（m/s）」という普遍的物理量の導入
競馬予想において最も広く使われるファクターは「走破タイム（時計）」ですが、走破タイムは距離ごとに基準が異なるため、前走1200mを走った馬と前走1400mを走った馬の履歴を単純比較できません。
そこで、距離の違いを規格化するため、過去2年間のレース記録から**「最高平均速度 (m/s)」**を直交特徴量として計算し、物理単位に落とし込みました。

$$v = \frac{Distance}{Time}$$

さらに、競馬において勝負を決定づけると言われる上がり3ハロン（最後の600mの走破タイム）についても、距離に依存しない純粋な「瞬発力・末脚（最終加速力）」の指数として抽出し、`horse_recent_avg_final_600m` としてモデルに追加しました。
これによりモデルは、過去の実績（順位データ）という間接的な結果だけでなく、馬の物理的なエンジンの出力性能から能力を客観的に推し量る能力を獲得しました。

## 3. Cycle 7: 状態変化指標（斤量増減フラグ）
馬体重と斤量について、オッズを形成する市場参加者（ファン）は「絶対値が大きいか小さいか」よりも「前走からどれだけ条件が変動したか」という相対的変化に非常に敏感に反応します。

* **`handicap_diff`**: 今回の斤量 - 前走の斤量。

「今回大幅に斤量が減った（恩恵により激走のチャンス）」や「前走より極端に斤量が増えた（ペナルティ）」という情報が、ファンの過剰な期待や失望につながりオッズを大きく変動させます。モデルにこの変化量を明示的に組み込むことで、物理的な有利不利と同時にファンの期待値の変動を予測する精度が向上しました。
